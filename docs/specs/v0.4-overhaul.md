## 0. TL;DR

- **新定位**：「**重度折腾派的本地 AI dotfiles 管家**」——npm 单包，跨 4 工具 × 3 资源类型，所有写入操作**改前 Diff + 密钥脱敏**，零云依赖。
- **杀手功能**：业界全空白的 **Diff/Preview 模态 + 密钥脱敏**。这是 v0.4 的护城河声明书。
- **技术栈**：React + Vite + Tailwind + Zustand（前端构建产物内联）；后端保留 `node:http` + 薄路由层 + zod；引入 ink 推迟到 v0.5。
- **范围**：5 个 Phase（P0 清债 → P1 数据结构 → P2 Diff Preview → P3 UI 重写 → P5 发布）。Ink TUI 砍掉。
- **时间**：约 6-7 周拿出 v0.4.0。
- **铁律**：`state.json` 自动迁移、归档不丢、`/api/v1` 完整保留至 v0.5。

## 1. 现状诊断（Linus 五层思考）

### 1.1 真问题，不是臆想

`skill-manager` 已经迭代到 v0.3.1，存在两份计划文档，但"不达预期"。基于业界调研与代码深度分析的交叉证据，根因有五层：

| 层 | 根因 | 关键证据 |
|---|---|---|
| **L1 数据结构** | `ResourceKind` 不是一等公民；三个 Record 平行设计；adapter 无统一 Port 接口 | `ResourceKind` 藏在 `src/core/state.ts:8-19`；`SkillRecord/SubagentRecord/McpServerRecord` 在 `src/types.ts` 三套平行 |
| **L2 特殊情况** | `codex` 撒谎、`agents` 命名三重歧义、`legacy.ts` 空壳、`listSkills` 死分支、两对 adapter 重复 | `src/core/list.ts:94-98, 159-162`；`src/cli/commands/legacy.ts:7-9`；`src/adapters/cursor.ts:13-37` ≈ `src/adapters/claude.ts:8-32` |
| **L3 复杂度** | 上帝文件 + 主路径 `innerHTML` 重写 + 535 行手写路由 | `src/ui/web/app.js` 1446 行；`src/ui/webapp.ts` 1854 行（生成）；`src/ui/server.ts` 535 行 |
| **L4 破坏性** | 计划与代码事实严重脱节，开发者不知信谁 | `STATE_VERSION = 4` (`src/core/state.ts:6`) vs v2-plan 写"版本 2"（`docs/skill-manager-v2-plan.md:179`）；IMPLEMENTATION_PLAN 阶段 1 标"未开始"但多项已落地 |
| **L5 实用性** | 缺杀手功能。业界**所有竞品**都没把"写入前 Diff（带密钥脱敏）"当核心 | `src/ui/web/app.js:848-850` `buildPreview` 注释明确"无服务端往返" |

### 1.2 业界格局（决定差异化）

| 赛道 | 锁定者 | 我们能否进入 |
|---|---|---|
| MCP 云目录站 | Smithery、Glama、PulseMCP | ❌ 已被锁定，不进 |
| MCP CLI 管理 | mcpm.sh | ❌ 单点已被占 |
| 单工具 skills 管理 | Arxchibobo（Claude）、vishalguptax（VSCode） | ❌ 已有玩家 |
| **跨工具 × 跨资源类型 × 本地 UI × 改前 Diff** | **业界全空白** | ✅ **这就是我们的位置** |

## 2. 新定位

> **「重度折腾派的本地 AI dotfiles 管家」**

### 2.1 三个支点（护城河）

1. **跨工具 × 跨资源类型**：Cursor + Claude Code + Codex + Agents，覆盖 Skills + Subagents + MCP servers。
2. **改前 Diff + 密钥脱敏**：所有 enable/disable 都先返回 unified diff，密钥脱敏，确认后才写盘。**业界全空白**。
3. **`npm i -g` 即用**：零云依赖，单包发布，重度用户的肌肉记忆。

### 2.2 目标用户

**Power User（重度折腾派）**：自己装 Cursor + Claude + Codex，关心 dotfiles、symlink、archive 这种细节，能接受信息密度高的专业 UI。

### 2.3 坚决不做的事（边界即护城河）

- ❌ 不做云市场 / skill 目录站
- ❌ 不追内置 skills/agents 数量
- ❌ 不复刻 Cursor 内置 Skills 面板
- ❌ 不做 VSCode 扩展形态（会失去 Codex CLI 用户）
- ❌ **不引入 Tauri/Electron**（npm 单包即用是核心护城河）

## 4. 新架构设计

### 4.1 领域模型（消除"特殊情况"）

`ResourceKind` 提为一等公民，三个 Record 收敛为 discriminated union：

```ts
// src/domain/types.ts
export type ResourceKind = "skill" | "subagent" | "mcp_server";
export type ToolId = "cursor" | "claude-code" | "codex" | "agents" | "vscode" | "codebuddy";

interface ResourceBase<K extends ResourceKind> {
  kind: K;
  tool: ToolId;
  id: string;
  displayName: string;
  description: string;
  sourceKind: "user-global" | "project" | "plugin";
  path: string;
  enabled: boolean;
  enabledSemantic: "native" | "managed";
  notes?: string;
}

export type Resource =
  | (ResourceBase<"skill"> & {
      invocation?: { disableModelInvocation?: boolean };
      skillKind: "markdown" | "cursor-builtin";
      pluginKey?: string;
    })
  | (ResourceBase<"subagent"> & { /* no extra */ })
  | (ResourceBase<"mcp_server"> & {
      transport: "stdio" | "http" | "unknown";
      command?: string;
      args?: string[];
      url?: string;
      envKeys: string[];  // 仅 key 名，值在服务端脱敏
    });
```

适配器统一为两个 Port：

```ts
export interface DiscoveryPort<K extends ResourceKind> {
  kind: K;
  scan(ctx: ScanContext): Promise<Extract<Resource, { kind: K }>[]>;
}

export interface MutationPort<K extends ResourceKind> {
  kind: K;
  preview(target: Resource, op: "enable" | "disable", strategy: Strategy): Promise<DiffPreview>;
  apply(target: Resource, op: "enable" | "disable", strategy: Strategy, opts: ApplyOptions): Promise<MutationResult>;
}

export interface DiffPreview {
  files: Array<{
    path: string;
    kind: "create" | "modify" | "delete" | "move";
    before?: string;  // 已脱敏
    after?: string;   // 已脱敏
  }>;
  unifiedDiff: string;       // 已脱敏的 unified diff
  redactedEnvKeys: string[]; // 被脱敏的 key 名（前端展示用 badge）
  warnings: string[];
}
```

这一刀下去：
- `asToolId/asSubagentToolId/asMcpToolId` 三处手写校验 → 收敛为一个 zod schema
- `cursor.ts/claude.ts` 的 `skillRecordFromDir` → 参数化工厂
- `mcp-cursor.ts/mcp-claude.ts` 的 `recordsFromFile` → 参数化工厂
- 任意 `if (kind === "skill") ... else if (kind === "subagent") ...` → discriminated union 收敛

### 4.2 仓库结构

```
src/
  domain/           # types + zod schema + ResourceKind 一等公民
  discovery/        # DiscoveryPort 实现，按 tool×kind 拆 adapter
  mutation/         # MutationPort 实现 + preview 引擎 + 密钥脱敏
    redact/         # 脱敏黑名单 + 单测
  state/            # state.json v1..v5 迁移 + archive + symlink 簿记
  config/           # config.yaml 加载/合并/校验
  ipc/              # 路由层 + SSE（CLI 和 UI 共用）
    routes/v1/      # 现有 API（保留至 v0.5）
    routes/v2/      # 新 API（统一 /resources）
  cli/              # commander + 子命令
  web/              # vite 项目源码（独立 src，构建产物内联到 webapp.ts）
scripts/
  build-web.ts      # vite build → 注入 WEBAPP_HTML
bin/skills-manager.mjs  # 不动
```

### 4.3 API v2 设计

```
GET  /api/v2/resources?kind=skill&tool=cursor    # 统一接口
POST /api/v2/resources/:kind/preview              # 新增：返回 DiffPreview
POST /api/v2/resources/:kind/apply                # enable/disable 统一入口
GET  /api/v2/events                               # SSE（保留 v1 行为）
GET  /api/v2/config + POST /api/v2/config         # 与 v1 一致
GET  /api/v2/doctor                               # 与 v1 一致
GET  /api/v2/fs/exists?path=...                   # 新增：补 v3 计划缺口
```

`/api/v1` 加 `Deprecation: true` + `Sunset: <v0.5 ETA>` header；v0.5 移除。

### 4.4 UI 信息架构

```
┌─────────────────────────────────────────────────────────────────┐
│ skills-manager · cwd: ~/projects/foo · doctor: ✓ · 🔍 Cmd+K     │
├──────────────┬──────────────────────────────────┬───────────────┤
│ Resources    │  Skills · 42 items · 12 enabled  │  Detail       │
│ ▼ Skills 42  │  ┌────────────────────────────┐  │               │
│   ▼ Cursor   │  │ ☐ name      tool    status │  │  YAML preview │
│     user 12  │  │ ☐ git-flow  cursor  on  ❯  │  │  + Diff if    │
│     proj  3  │  │ ☐ test-pyr  claude  off    │  │    pending    │
│   ▼ Claude   │  │ ...                        │  │               │
│ ▼ Subagents  │  └────────────────────────────┘  │  [Enable]     │
│ ▼ MCP        │  [Bulk: Enable | Disable | Add  │  [Disable]     │
│ ─────────    │   to unified.select]            │  [Preview...]  │
│ Filters      │                                  │               │
└──────────────┴──────────────────────────────────┴───────────────┘
```

**核心 UX 元素**：
- 三栏布局：左资源树 + 中列表（虚拟滚动 + 复选 + 批量条）+ 右详情/Diff
- **Cmd+K 命令面板**：跳转资源、切 tool、触发批量操作
- **Diff Modal**：所有写入前必弹 unified diff（语法高亮 + 密钥脱敏 badge）
- **底部批量条**：显示选中数和待执行操作
- **顶部状态条**：cwd、doctor 状态、SSE 连接、版本号
- **三态完整**：空态/loading/错误态，错误带可复制堆栈

### 4.5 技术栈决策矩阵

| 层 | 选型 | 理由 |
|---|---|---|
| 前端框架 | React + Vite | 重度用户期望专业 UI；React 在 IDE 内嵌场景生态最强 |
| 状态管理 | Zustand | 轻量、TypeScript 友好、无 Provider 嵌套 |
| 样式 | Tailwind CSS（构建时 purge） | 高信息密度场景适配；产物 < 50 KB |
| Diff 视图 | react-diff-viewer（备选 monaco-diff） | 产物体积可控；monaco 太大 |
| 命令面板 | kbar（或自实现） | 体积小、键位定制强 |
| 后端路由 | `node:http` + 60 行自实现路由 | 保持零运行时 dep |
| 校验 | zod | 运行时 + 类型双重派生；替代三处手写守卫 |
| 测试 | vitest（保留）+ Playwright（新增） | E2E 是 v0.4 验收关键 |

### 4.6 运行时依赖增加

**进 `dependencies`**：
- `zod`（约 50 KB）

**进 `devDependencies`** 但**不进运行时**（vite build 内联到 webapp.ts）：
- `react`, `react-dom`
- `zustand`
- `tailwindcss`, `postcss`, `autoprefixer`
- `react-diff-viewer-continued`
- `kbar`
- `vite`

**前端构建产物预算**：gzip 后 ≤ 180 KB（强约束，超标则砍 kbar 换简易实现）。

### Phase 2 — 杀手功能：Diff/Preview API + 密钥脱敏（1.5 周）

**目标**：业界全空白的护城河功能落地。这是 v0.4 的定位声明书。

**任务**：
1. 在 `src/mutation/` 实现 `MutationPort.preview(target, op, strategy)` 返回 `DiffPreview`：
   - **Skill native**：模拟修改 frontmatter，diff 整个文件
   - **Skill managed**：模拟 archive 移动，diff 输出"move" 操作描述
   - **Skill symlink**：模拟创建/删除 symlink
   - **Subagent native/managed**：同 skill
   - **MCP enable/disable**：模拟修改 `mcp.json` / `~/.claude.json`，diff JSON
   - **MCP symlink**：模拟 unified 目录变更
2. 写入引擎纯函数化：`generateNewContent(target, op, strategy) → string | FileOp[]` → `diff(before, after) → UnifiedDiff`。
3. **密钥脱敏**（`src/mutation/redact/`）：
   - 黑名单字段名（正则）：`/(api[_-]?key|token|secret|password|authorization|bearer)/i`
   - 黑名单值正则：`/(sk-[A-Za-z0-9]+|ghp_[A-Za-z0-9]+|...)`
   - 通用兜底：长度 ≥ 32 的 base64/hex 串可选脱敏
   - 脱敏后用 `***` 替换，原 key 名加入 `redactedEnvKeys`
4. 新增 `/api/v2/resources/:kind/preview` 路由。
5. CLI 新增 `--preview` 标志：`skills-manager skills disable foo --preview` 打印 diff 但不写。
6. 单测覆盖：所有 3 kind × 4 strategy × 2 op = 24 个组合至少各一个测试。

**完成标准**：
- 100 次 preview 调用后 `state.json`、archive、目标文件零变化
- `OPENAI_API_KEY=sk-xxxxx` 在 diff 中显示为 `***` + `redactedEnvKeys` 含 `OPENAI_API_KEY`
- 手工：`skills-manager mcp disable github --preview` 看到正确 diff

**测试用例**：
- `__tests__/mutation/preview.test.ts`：24 组合 × 副作用为 0 断言
- `__tests__/mutation/redact.test.ts`：常见密钥 pattern 全部脱敏
- `__tests__/mutation/redact.test.ts`：不应脱敏的 path/url/普通字符串不误伤
- E2E：CI 加 `gitleaks` 扫示例 fixture 文件

**回滚**：纯增量，无破坏性。

### Phase 3 — UI 重写（主线，2.5-3 周）

**目标**：把 1446 行 `app.js` 替换为模块化 React 应用，主路径无 `innerHTML` 重写。

**任务**：
1. 新建 `src/web/` Vite + React + Tailwind + Zustand 项目骨架。
2. Zustand store 设计：
   - `resourceStore`：列表 + 筛选 + 选中
   - `mutationStore`：pending preview + 写入队列 + 错误
   - `configStore`：表单/JSON 双向同步
   - `eventStore`：SSE 订阅
3. 组件层：
   - `<ResourceTree>` 左栏（工具×kind×source 多级折叠）
   - `<ResourceTable>` 中栏（虚拟滚动 + 复选 + 排序）
   - `<DetailPanel>` 右栏（YAML preview + 操作按钮）
   - `<DiffModal>` 接 `/api/v2/resources/:kind/preview`（react-diff-viewer + 密钥 badge）
   - `<BulkActionBar>` 底部固定
   - `<CommandPalette>` Cmd+K（kbar 或自实现）
   - `<ConfigEditor>` 表单 ⇄ JSON 双向 + zod schema 错误高亮 + `extraRoots` 拖拽 + 路径校验（接 `/api/v2/fs/exists`）
4. SSE 接入 `eventStore`，节流 200ms。
5. `scripts/build-web.ts` 改造：
   - `vite build` → 单 HTML（含 inline JS+CSS）
   - 注入 `WEBAPP_HTML` 模板字符串
   - CI 校验产物 gzip 后 ≤ 180 KB
